原型模式：（创造型模式）
原型实例制定创建对象的种类，并且通过拷贝这些原型创建新的对象（不需要知道创建细节，不用构造方法）
应用场景：
1.类初始化消耗资源较多
2.new产生的一个对象需要非常繁琐的过程（数据准备，访问权限等）
3.构造函数比较复杂
4.循环体中生产大量对象时，可读性差
